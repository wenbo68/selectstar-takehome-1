# 사전 과제

### 프로젝트 과제 안내

본 과제는 코드 구현 과제가 아니며 **실제 서비스 또는 데이터 파이프라인에 적용 가능한 수준의 Reasoning 생성 및 질문 중복 제거 프로세스를 설계하는 것을 목표로 합니다.**

단순 구현 능력보다, **설계 전략·프롬프트 구성·Trade-off 분석·고도화 방향 제시** 등 전문적인 사고 과정을 평가합니다.

정답이 존재하는 형태가 아니며, 설계 방식과 근거의 논리성을 중점적으로 평가합니다.

### 제출 형식

- PDF 또는 PPT 형식의 문서
- 제출 및 문의 이메일: `AI_innov@selectstar.ai`
- 코드 구현은 필수가 아닙니다.
- 파이프라인 흐름도 또는 구성도 포함 시 가산점

### 평가 기준

- 설계 과정의 논리성 및 분석의 타당성
- 파이프라인 구성 요소가 명확히 정의되었는가
- 비용–품질–시간 trade-off 분석 수준
- 프롬프트/에이전트 설계의 구체성 및 실무적 확장 가능성

### 과제

**목표**

수학 문제에 대한 Q와 A가 주어집니다. 주어진 Q/A를 활용하여 고품질 Reasoning을 생성하고, Q/A 데이터를 분석하여 중복 Q를 제거하는 파이프라인을 설계합니다. 이 때, 반드시 임베딩 기반 중복 제거 파이프라인을 포함시킵니다. 이렇게 중복제거된 데이터는 모델에 수학 지식을 높이기 위한 학습용 데이터로 사용됩니다.

단순히 모델을 호출하여 결과를 얻는 수준이 아니라, 프롬프트 설계, Reasoning 생성/품질 제어/검수 전략, 임베딩 전략, 중복 질문 정의 및 제거 기준, 비용/품질/시간의 트레이드 오프 등을 고려한 실무형 설계가 핵심입니다. 위 항목은 예시이며 반드시 포함할 필요는 없습니다.

**제공 데이터**

- 수학 Q/A 데이터 130건
- 제공 데이터는 설계 참고 및 데이터 분석을 위한 예시입니다.
- 제공 데이터는 SelectStar 소유이며 외부 유출을 금합니다.

**필수 수행 사항**

- 각 기능별 모델 선정 및 선정 이유
- 프롬프트 / 에이전트 설계
- 비용 / 품질 / 시간 관점에서 파이프라인 전략 분석
- (Optional) 고도화 제안


### example Q/A pairs

1. 
- Q:
"$a, b, c$를 양의 실수라고 하자.  
다음 세 다항식  
$$
a x^{2} + b x + c, \quad b x^{2} + c x + a, \quad c x^{2} + a x + b
$$  
가 가질 수 있는 실근의 총 개수의 최댓값을 구하여라."
- A:
4

2. 
- Q:
"적분 
$$
\int_{0}^{\frac{\pi}{2}} \frac{x}{\sin x + \cos x}\, dx
$$
의 값을 구하여라."
- A:
\(\frac{\pi\sqrt{2}}{4} \ln (\sqrt{2}+1)\)

3. 
- Q:
"\(0 < x < y \le z \le 1\) 이고 \(3x + 2y + z \le 4\) 일 때,  
\(P = 3x^2 + 2y^2 + z^2\) 의 최댓값을 구하여라."
- A:
10/3
